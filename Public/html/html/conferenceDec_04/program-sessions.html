Wednesday, Dec 8th  8:30-11:30 a.m.
Pre-Conference DG Content & Authoring

The work of the Sakai Content Management (CM) Service Requirements team will be reviewed in this public session.

Wednesday, Dec. 8th 1:30-4:30 p.m.
Pre-Conference: Building a Sakai Tool

This three-hour workshop will cover the design and development of Sakai tools and services.  It will include a discussion on how to create new APIs, implementation application services, how to manage service dependencies, using JavaServer Faces to develop a user interface, framework considerations and services available to the developer, debugging and development techniques and much more.  The presentation is very focused on providing the technical information needed to develop for Sakai and is targeted at Java programmers.  While some of the information presented at the first conference will be shown again, there is a great deal of new material in this workshop.

Mark Norton, SEPP Senior Technical Consultant
 

Wednesday, Dec. 8th 5:30-7:30 p.m.

Welcome & Sakai Project Overview

The Sakai Project is almost a year old now and has an impressive set of accomplishments to look back on.  We also have a daunting set of tasks before us.  This session will give an overview of our work to date, and a look ahead for the coming year.

What’s new for Sakai 1.5?  A Feature Demonstration

Rob Lowden, Indiana University

Thursday, Dec. 9th 10:45–12:30 p.m.
Sakai Architecture Update

This talk will describe the Sakai Architecture as described in the Sakai Java Framework Document.  We will highlight changes in the framework and architecture since the previous SEPP meeting.   We will describe the new changes for the 1.5 release both from a tool and framework Other topics include:  IMS Tool Interoperability effort, iFrame integration, WSRP plans and portal integration plans.  In addition there will be some discussion of Framework I (legacy) and Framework II issues.  The talk will also include a discussion regarding the general direction for the 2.0 release, possible features, and framework changes.

Charles Severance, University of Michigan
 

Thursday, Dec. 9th 10:45–12:30 p.m.

Sakai Tools Update

This session will include a detailed review and presentation of the current timeline of events for design and feature development, where we are in that timeline and an overview of where we anticipate being in the future.  Updated Style Guide information will be presented in conjunction with a demonstration of the latest Style Guide version, access to the resource and explanations of intended audience and use.  Examples will be provided on how one should ideally interact with the style guide, use case examples from the TTeam representative’s experiences.  The presentation will conclude with a detailed presentation of how the design, development, deployment, bug tracking and QA process for Sakai will be implemented and ultimately opened to the Sakai community.     

Rob Lowden, Indiana University

DG Development

The theme of the discussion will be "Simplifying Sakai Development".  I'd like to explore some of the things that make developing Sakai tools and services difficult and explore potential solutions to be fed back to the Sakai Architecture Team.  Do we need more logging?  Are the services difficult to find and understand?  How is the documentation?  What are the common development problems?  I would especially like to invite developers with Sakai development experience to participate and share the things they've learned.

The guiding principles, goals, and rationale behind development decisions will be presented along their impact on migrating thousands of California Community College users from ETUDES to ETUDES-NG (local Sakai brand), starting fall 2005.

DG User Interface

We hope to launch the SEPP User Interface discussion group (UI DG).  The DG's charter statement, describing its initial direction, reads: "The SEPP User Interface discussion group (UI DG) is intended to be a forum to identify, discuss, and debate issues and opportunities that relate to Sakai user interface issues.  Although UI DG is a broad and open form, its particular interest concerns user interface issues from the end user perspective. Within that context, the UI DG can address areas such as accessibility, universal design, adherence to relevant standards, as well as other areas as needed.

DG Cross Language

During hour one of the Cross Language Discussion Group meeting, Chuck Severance will explain the role of WSRP in the Sakai framework and then take questions.  Two key points that will emerge from Chuck's discussion are: (1) any portal that has a WSRP consumer can use Sakai tools, and (2) any portal that supports iFrames can also use Sakai tools directly via HTTP.  In light of this, during hour two we will focus on four integration scenarios outlined in the cross language survey, and the services required to pursue them. The end goal of the session is to identify pilot integration projects.

Tom Lewis, University of Washington

DG Strategy & Advocacy – Governance I

This will be a working session to identify what participants think should to be in place in order to make SAKAI/SEPP sustainable and successful for the future. The goal of all of this is to build a 'straw man' governance proposal which can be discussed online by the larger community.  Ultimately the items arising from Governance I may be used as input to the Board or other working groups, but the focus is on a governance model rather that a task list.  I'm hoping that the end result of all of this will be a series of recommendations endorsed by both the SAKAI Board members and the SEPP members.

Lionel Tolan, Simon Fraser University

DG Library & Repositories II

Sakai / Digital Repository Connectivity -  Library Services

Another major focus of the Library and Repositories group is the delivery of library services into course sites in Sakai. Some of the innovation will come through the creation of Sakai tools. One current area of interest, for example, is the development of tools capable of federated searching of disparate catalogs and repositories. Other possibilities might entail refactoring the delivery of current library services to leverage existing tools. An example of this would be to expose electronic reserves lists using RSS, taking advantage of the Sakai News tool. These examples just scratch the surface. We will discuss some of the underlying technologies and research projects in this important area.

Bill Parod, Northwestern University
Jim Martino, Johns Hopkins University

DG Content & Authoring - Pedagogy

Sakai now has the basic “tools” that are expected in a course management system. Basic tools are important because users expect them and user expectations must be satisfied to have widespread adoption of Sakai.  But now as the basic tools are almost ready, we must think about what other functionality is needed to support learning; what other tools are needed for a next generation Collaboration Learning Environment. We in the Sakai Project can take several approaches to influence web-supported teaching and learning. This presentation will discuss some of the possibilities.

Charles Kerns, Stanford University
