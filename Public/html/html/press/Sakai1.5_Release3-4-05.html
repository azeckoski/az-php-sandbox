Press Release: March 7, 2005

Sakai Project releases version 1.5 of its Collaboration and Learning environment (CLE) software

ANN ARBOR, Mich.---On March 4, 2005, the Sakai Project released version 1.5 of its Collaboration and Learning environment (CLE) software, with many improvements for faculty, students, and technical support staff. The long list of changes includes new tools, new capabilities in existing features, system improvements, and bug fixes.

Sakai 1.5 has already been in use at IU since the beginning of January, and there has been extensive quality and assurance testing by the Sakai community. Faculty reactions to the new software point to encouraged optimism  --they like what they see in the practicality of the teaching tools and look forward to more tools in future versions. Robert Appelman, IU Professor of Education, has been teaching with Sakai 1.5.  He observed that "Sakai offers me the opportunity now to have a course working environment that is much more integrated than before.  This integration specifically allows linking and gaining access to resources and methods used in other classes that before were possible, but simply impractical."

This release is Sakai's first since the Educational Community License (ECL) became an Open Source Initiative (OSI) approved license. "We are 14 months into the Community Source Sakai Project, and this release marks a major milestone in maturing both the software and the community processes for its long term evolution," said Joseph Hardin, Chairman of the Sakai Project Board.

Among the Sakai Core Institutions and the 68 members of the Sakai Educational Partners Program (SEPP), there are dozens of pilots underway. Scott Siddall with the Longsight Group reports work with seven smaller colleges in full pilot with the 1.5 code as it was going through refinement in January and February.  "The ability to track the builds, bugs, fixes, and release candidates has been amazing as we have supported several colleges with their Sakai 1.5 pilots," said Siddall. See the Sakai Adoption and Exploration page for a complete of Sakai pilots.

NEW TOOLS IN SAKAI 1.5

  • Syllabus Tool -- implementation of the Syllabus Tool, which allows faculty to display their department's official online syllabus.
  • SAMIGO Assessment Tool -- Partial integration of the IM QTI-based Quizzing and Testing Tool, which provides:
    • - assessment authoring
    • - assessment publishing
    • - assessment taking
    • - assessment grading
    • - question pool management
    • - template management
  • Contextual Help Tool -- Question marks (?) in the tools throughout the software that take users to sections of the online help with explanations of the tool.

NEW CAPABILITIES IN EXISTING FEATURES

  • The ability for users to disable auto-refresh.
  • In the Assignment Tool, the addition of Grader Permission, which can be turned on or off for each role.
  • Database conversion tool -- A conversion tool to get items that were created in Sakai 1.0 converted to work in Sakai 1.5.
  • More options for flagging copyright materials.
  • URLs that better reflect site content.
  • Better compliance with the Sakai Style Guide.
  • The ability for adopters to configure whether resources will be stored in a database or file system server.
  • The ability for an adopter to enable User Presence in the UI (User Presence is disabled by default).
  • Calendar Date Entry Widget
  • WYSIWYG Editor
  • Improvements to support reporting functions including the ability to identify current user sessions.
  • Foreign Language Support for non-Roman characters  (At present, characters can only be entered from left to right.)
  • Export/Import tool for sites as part of the Worksite Setup and Site Info tools. This tool allows moving content from one site to another.

SYSTEM IMPROVEMENTS

  • Scalability - Sakai 1.5 is much more scalable than the 1.0 release as we have reduced the memory footprint and normalized many objects into a relational database.
  • Improvements in database performance
  • Improved JSF support
  • Established HSQL as the default database
  • Improved Realm APIs
  • Improved Site Service APIs
